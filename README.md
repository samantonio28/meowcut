# MeowCut — сервис укорачивания ссылок

Сервис предоставляет API для создания коротких ссылок и получения оригинальных URL.

## Требования

- Go 1.25.3+
- Docker и Docker Compose (для запуска с PostgreSQL)

## Структура проекта

```
.
├── cmd/                 # Точка входа приложения
├── configs/             # Конфигурационные файлы
├── internal/
│   ├── domain/         # Доменные сущности, интерфейсы, ошибки
│   ├── service/        # Реализации репозиториев и укорачивателя
│   ├── usecase/        # Бизнес-логика (создание и получение ссылок)
│   └── delivery/       # HTTP-обработчики и инициализация приложения
├── pkg/utils/          # Вспомогательные утилиты (генерация, валидация)
├── migrations/         # SQL-миграции для PostgreSQL
├── logs/               # Директория для логов
├── Dockerfile
├── docker-compose.yml
├── doc.yaml            # OpenAPI спецификация
├── generate.go         # Генерация кода из спецификации
└── README.md
```

## Конфигурация

### Файлы конфигурации

- `configs/server.yaml` — настройки сервера и тип хранилища.
- `configs/database.yaml` — параметры подключения к PostgreSQL.

### Переменные окружения

- `STORAGE_TYPE` — тип хранилища (`native` или `pg`). Переопределяет значение из конфига.
- `CONFIG_PATH` — путь к конфигурационному файлу (по умолчанию `configs/server.yaml`).

## Запуск

### Локальный запуск (с нативным хранилищем)

1. Установите зависимости:
   ```bash
   go mod download
   ```

2. Сгенерируйте код из OpenAPI спецификации:
   ```bash
   go generate ./...
   ```

3. Запустите сервер:
   ```bash
   go run ./cmd
   ```

### Запуск с PostgreSQL через Docker Compose

Соберите и запустите контейнеры:
```bash
docker-compose up --build
```

Сервис будет доступен на `http://localhost:8080`

### Запуск тестов

```bash
# Запуск всех тестов
make

# Запуск тестов usecase
make usecase

# Запуск тестов service
make service
```

## API

Спецификация OpenAPI доступна в файле `doc.yaml`. Сгенерированная документация может быть просмотрена с помощью Swagger UI.

## Генерация коротких ссылок

Используется случайная генерация строки длиной 10 символов из алфавита `[A-Za-z0-9_]`. Вероятность коллизии крайне мала (аналогично UUID). Для одного оригинального URL всегда возвращается один и тот же shortId (связь 1:1).

## Хранилища

Поддерживаются два типа хранилищ:

1. **In‑memory (native)** — хранение в map внутри приложения. Подходит для разработки и тестирования.
2. **PostgreSQL (pg)** — хранение в таблице `links`. Обеспечивает персистентность и возможность масштабирования.

Выбор хранилища задаётся в конфигурации или через переменную окружения `STORAGE_TYPE`.
